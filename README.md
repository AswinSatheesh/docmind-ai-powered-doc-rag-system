Users register and upload PDF documents. The system extracts text, chunks it into overlapping segments, generates embeddings via OpenAI, and stores them in a PostgreSQL vector index (PGVector). When a user asks a question, the system embeds the query, runs cosine similarity search to retrieve the top-5 relevant chunks, and sends them as context to an LLM to generate a grounded answer. Chat history is persisted per session. Kafka asynchronously processes document ingestion. Redis caches frequent document lookups and rate-limits the AI API calls.
